# NixOS Flake Installation Guide

Step-by-step guide for installing NixOS from a live USB/CD using this flake config.

---

## Prerequisites

- **NixOS live ISO** booted (nixos-unstable recommended for RX 9060 XT support)
- **Internet connection** (Wi-Fi or Ethernet)
- This repo pushed to GitHub at `github:Overload404/nix-migration`
- Your actual disk device name (examples here use `/dev/nvme0n1` — **replace with yours**)

---

## Step 1: Boot the Live ISO

Boot from the NixOS live USB. Once at the shell prompt, connect to the internet:

**Wired (Ethernet):** usually works out of the box — verify with `ping -c 3 google.com`.

**Wireless (Wi-Fi):**
```bash
# Enter wpa_supplicant shell
wpa_cli
> add_network
> set_network 0 ssid "YOUR_SSID"
> set_network 0 psk "YOUR_PASSWORD"
> enable_network 0
> quit
```

Verify connectivity:
```bash
ping -c 3 google.com
```

## Step 2: Partition the Disk

### 2.1 Identify your disk

```bash
lsblk
```

Look for your target disk (e.g. `nvme0n1`, `sda`, `vda`). **The rest of this guide uses `/dev/nvme0n1` — substitute your disk everywhere.**

### 2.2 Create partitions

Using `parted` (non-interactive, easy to script):

```bash
# Wipe existing partition table (DESTRUCTIVE — double-check the disk!)
parted /dev/nvme0n1 -- mklabel gpt

# Create EFI system partition (512 MiB)
parted /dev/nvme0n1 -- mkpart ESP fat32 1MiB 513MiB
parted /dev/nvme0n1 -- set 1 esp on

# Create root partition (rest of the disk)
parted /dev/nvme0n1 -- mkpart primary ext4 513MiB 100%
```

> **Alternative — using `fdisk`:** if you prefer interactive partitioning, run `fdisk /dev/nvme0n1` and create a GPT table, a 512M EFI partition (type 1), and a root partition (type 20).

### 2.3 Format the partitions

```bash
# EFI partition (must be FAT32)
mkfs.fat -F 32 /dev/nvme0n1p1

# Root partition (ext4)
mkfs.ext4 /dev/nvme0n1p2
```

> **Note on partition numbering:** NVMe drives use `p1`, `p2` suffixes. SATA drives (e.g. `/dev/sda`) use `1`, `2` (no `p`). Adjust accordingly.

### 2.4 Mount the filesystems

```bash
# Mount root first
mount /dev/nvme0n1p2 /mnt

# Create boot directory and mount EFI
mkdir -p /mnt/boot
mount /dev/nvme0n1p1 /mnt/boot
```

## Step 3: Clone the Config (Required for hardware-configuration.nix)

> **Why clone?** `nixos-install --flake github:...` would work for a pure remote flake, but our configuration references `./hardware-configuration.nix` — a machine-specific file generated by `nixos-generate-config`. This file is **never committed** to the repo (it's in `.gitignore`). Instead, the clone places the flake code in `/mnt/etc/nixos/`, then `nixos-generate-config --root /mnt` creates the hardware config right alongside it. Both need to be in the same directory.

```bash
# Get git (not included in minimal ISO)
nix-shell -p git

# Clone from GitHub
git clone https://github.com/Overload404/nix-migration.git /mnt/etc/nixos

# Generate hardware config (puts hardware-configuration.nix in /mnt/etc/nixos/)
nixos-generate-config --root /mnt
```

---

## Step 4: Install NixOS

```bash
nixos-install --flake /mnt/etc/nixos#overrig
```

Set root password when prompted. Then set user password:

```bash
nixos-enter --root /mnt -c 'passwd overload'
```

---

## Step 5: Reboot

```bash
reboot
```

Remove the USB drive when prompted (or when the system powers off). The system should boot into your new NixOS installation.

> If boot fails with a GRUB error, reboot from the live USB, re-mount, `nixos-enter`, and check `/boot` contents. The flake's `boot.loader` settings should handle this, but verify your EFI partition was mounted correctly.

---

## Step 6: Post-Install (from within NixOS)

Log in as your user (`overload`).

### 6.1 Verify Home Manager

Home Manager should already be activated from the flake install. Verify:

```bash
home-manager generations
```

If not, run:

```bash
home-manager switch --flake /etc/nixos#overload
```

### 6.2 Change your password

```bash
passwd
```

### 6.3 Set up wallpapers

```bash
mkdir -p ~/.wallpapers
# Copy your wallpapers into this directory
```

### 6.4 Copy personal configs

```bash
# Neovim (LazyVim)
cp -r /path/to/backup/.config/nvim ~/.config/nvim

# VPN keys
mkdir -p ~/.openvpn
cp /path/to/backup/openvpn/* ~/.openvpn/
```

### 6.5 Copy game/WoW directories (if applicable)

```bash
# Example — adjust paths to your backup location
cp -r /mnt/backup/games/wow ~/Games/wow
```

### 6.6 Rebuild to apply any changes

```bash
sudo nixos-rebuild switch --flake /etc/nixos#overrig
```

## Troubleshooting

### Brave: hash mismatch

If the Brave browser derivation fails with a hash mismatch, update the fetch:

```bash
nix-prefetch-url https://github.com/brave/brave-browser/releases/download/v<VERSION>/brave-browser-<VERSION>-1.x86_64.rpm
```

Then update the hash in your flake inputs or overlay.

### Gruvbox GTK themes not found

The `gruvbox-gtk-theme` package may not be in nixpkgs. Install it manually:

```bash
git clone https://github.com/Fausto-Korpsvart/Gruvbox-GTK-Theme ~/.themes/Gruvbox
```

Then point your GTK config at `~/.themes/Gruvbox`.

Or switch to a nixpkgs-available dark theme (e.g. `arc-theme`, `adwaita-dark`).

### Hyprland polkit agent fails

If the `hyprpolkitagent` fails to start or authenticate, switch to the GNOME polkit agent:

In your Hyprland config, replace the polkit agent exec line with:

```
exec-once = /run/current-system/sw/libexec/polkit-gnome-authentication-agent-1
```

Or set `systemd.user.services.polkit-gnome-authentication-agent-1` in your Home Manager config.

### General: "command not found" after install

Your user shell may not have loaded the Nix profile yet. Log out and back in, or run:

```bash
source /etc/set-environment
```

### General: Flake build fails with missing inputs

Ensure your flake inputs are accessible:

```bash
nix flake lock --update-input nixpkgs
nix flake lock --update-input home-manager
```

Then retry the build.

## Quick Reference: Disk Commands

| Action | NVMe example | SATA example |
|--------|-------------|--------------|
| List disks | `lsblk` | `lsblk` |
| EFI partition | `/dev/nvme0n1p1` | `/dev/sda1` |
| Root partition | `/dev/nvme0n1p2` | `/dev/sda2` |
| Format EFI | `mkfs.fat -F 32 /dev/nvme0n1p1` | `mkfs.fat -F 32 /dev/sda1` |
| Format root | `mkfs.ext4 /dev/nvme0n1p2` | `mkfs.ext4 /dev/sda2` |
| Mount root | `mount /dev/nvme0n1p2 /mnt` | `mount /dev/sda2 /mnt` |
| Mount EFI | `mount /dev/nvme0n1p1 /mnt/boot` | `mount /dev/sda1 /mnt/boot` |

---

**Remember:** every `/dev/nvme0n1` in this guide is an example — **replace it with your actual disk** (check with `lsblk`).
